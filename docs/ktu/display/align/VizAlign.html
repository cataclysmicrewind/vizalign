Classpublic class VizAlign
InheritanceVizAlign AlignMethods AlignUtils

Duplicates all of the align panel functions you see inside the Flash IDE and more. Added functionality allows you to align any number of objects to another (like toStage, but with any other display object). There are 33 alignment methods to choose from, some of them may seem redundant, but choosing them may save you CPU time... VizAlign has all the functionality of the Flash IDE align panel. All of your standard alignment methods are here plus a bunch of other useful ones.


Example
This will align the two objects to the left edge of the bounding area the targets create.
With two objects on the stage labeled mc1, and mc2, call this code: 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var param:AlignParam = new AlignParam( VizAlign.LEFT, VizAlign.TO_TARGETS );
    VizAlign.align( [mc1, mc2], [ param ], true, stage );


This will align the two objects to the right edge of the stage.
With two objects on the stage labeled mc1, and mc2, call this code: 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var param:AlignParam = new AlignParam( VizAlign.RIGHT, stage );
    VizAlign.align( [mc1, mc2], [ param ], true, stage );



Lets say you want to align your targets to space evenly along the stage, and then to align
bottom to the bounding area the targets create: (you need four objects from mc1...mc4) 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var param1:AlignParam = new AlignParam( VizAlign.SPACE_HORIZONTALLY, stage);
    var param1:AlignParam = new AlignParam( VizAlign.BOTTOM, VizAlign.TO_TARGETS);
   VizAlign.align( [mc1, mc2, mc3, mc4], [ param1, param2 ], true, stage );




Simple: Align targets, to one targetCoordinateSpace
Multiple: Align targets, with multiple methods
Groups: Align groups of targets, with any of the above.

Simple: 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var param:AlignParam = new AlignParam (VizAlign.LEFT, VizAlign.TO_TARGETS);
    VizAlign.align( [mc1, mc2], [ param ], true, stage );


Multiple: 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var param1:AlignParam = new AlignParam (VizAlign.LEFT, VizAlign.TO_TARGETS);
    var param2:AlignParam = new AlignParam (VizAlign.TOP, VizAlign.TO_TARGETS);
    VizAlign.align( [mc1, mc2], [ param1, param2 ], true );



Groups: 
    import com.crp.display.utils.VizAlign;
    import com.crp.display.utils.AlignParam;                           

    var group1:Array = [mc1, mc2];
    var group2:Array = [mc3, mc4];
    var group3:Array = [mc5, mc6, mc7];
    var param:AlignParam = new AlignParam (VizAlign.LEFT, VizAlign.TO_TARGETS);
    VizAlign.align( [ group1, group2, group3 ], [ param ], true, stage );

To treat multiple targets as a group, pass an array of each grouping as an element of the targets paramater. Return array for this example would be: results[0] is an Array of AlignObjects, with the targets for the first group, and so on.

Property detail
truePixelproperty
truePixel:Boolean  [read-write]

Specifies whether to round the end results of alignment

The default value is true When truePixel = true, x,y,width, and height values are rounded using Math.round(); This is suggested and the default because when DisplayObject positioning end up on anything but an int, visuals can become blurred as Flash Player will try to blend colors. When TextField end up on a partial pixel, the anti ailiasing becomes poor..

Returns
